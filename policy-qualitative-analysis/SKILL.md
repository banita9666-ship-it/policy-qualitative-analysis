# 政策文本质性分析

基于公共管理与政策科学成熟方法论，对政策文本进行系统性质性分析。

## 分析流程概览

1. 明确研究问题与分析目标
2. 收集与筛选政策文本
3. 选择分析方法（质性/量化/混合）
4. 构建分析框架
5. 编码与信效度检验
6. 多维度分析
7. 撰写分析报告

## 第一步：研究设计

向用户确认以下要素：

- **研究问题**：要解决什么政策问题？
- **政策领域**：教育、医疗、环保、科技、文化等？
- **时间范围**：哪些年份？
- **行政层级**：中央/省级/市级？
- **分析目的**：描述性、评价性、比较性、理论构建？
- **政策类型**：法律、法规、意见、通知、规划、方案？

**参考 `references/政策类型适配.md`** 了解不同政策类型的分析要点。

## 第二步：文本收集

### 数据来源
- 官方数据库：北大法宝、国家法律法规数据库、国务院政策文件库、各部委官网
- 政策类型：法律、行政法规、部门规章、意见、通知、规划、方案、决定

### 筛选流程
1. 以关键词+领域进行初步检索
2. 按政策类型筛选（排除批复、函件、非规范性文件）
3. 按时效筛选（排除已废止或被替代的政策）
4. 按权威性筛选（优先选择高级别发文）
5. 形成最终样本并说明纳入/排除理由

## 第三步：选择分析方法

**阅读 `references/分析方法体系.md` 获取各方法的详细说明。**

根据研究问题选择方法：

| 研究问题类型 | 推荐方法 |
|---|---|
| 政策文本蕴含什么意义/意图？ | 质性：释义分析、话语分析 |
| 政策如何随时间演变？ | 量化：文献计量+趋势分析 |
| 政策使用了哪些工具/机制？ | 量化：内容分析+政策工具框架 |
| 政策是否有效/质量如何？ | 量化：PMC指数模型评价 |
| 从中能提炼出哪些主题/范畴？ | 质性：扎根理论（三级编码） |
| 政策间或地区间如何比较？ | 质性：比较分析+量化指标 |
| 政策话语如何建构问题？ | 质性：叙事政策框架、批判话语分析 |
| 政策主体间如何互动？ | 量化：社会网络分析 |
| 大规模政策语料有何规律？ | 量化：文本挖掘（LDA/BERT） |

**推荐**：综合研究采用混合方法，质性解读与量化内容分析相结合。

## 第四步：构建分析框架

**阅读 `references/分析框架模板.md` 获取即用型框架。**

### 常用框架类型

**A. 政策工具框架（最常用）**
- X维：政策工具（供给型/环境型/需求型 或 强制型/混合型/自愿型）
- Y维：政策目标（领域特定）
- Z维：政策主体（政府部门、市场主体、公众等）

**B. PMC指数模型（政策评价）**
- 9个一级变量（政策性质、时效性、目标、工具、对象、内容、领域、保障、评估）
- 二级变量二值评分（0/1）
- PMC指数计算与评级

**C. 扎根理论框架**
- 开放式编码 → 主轴编码 → 选择性编码
- 从数据中建构理论模型

**D. 自定义框架**
- 基于文献综述+研究问题推导维度
- 确保互斥性与完备性

## 第五步：编码与信效度检验

**阅读 `references/编码与信效度.md` 获取详细操作规范。**

### 编码规则
- 分析单元：政策条款/句子（不可再分原则）
- 编码格式：`政策编号-章-节-条`（如 `1-3-2-1`）
- 一个单元可在不同维度上获得多个编码
- 编码至最细子类别层级

### 信效度检验
- 至少2名独立编码员
- 计算编码者间一致性（Kappa系数或百分比一致率）
- 可接受阈值：Kappa ≥ 0.8 或一致率 ≥ 85%
- 分歧通过讨论或第三方仲裁解决

## 第六步：开展分析

### 单维度分析
- 各类别频次统计与百分比分布
- 时间趋势可视化
- 识别结构性失衡

### 交叉维度分析
- 交叉表分析（如 工具×目标、工具×主体）
- 识别维度间错配
- 热力图或网络图可视化
- 揭示政策模式与偏好

### 典型分析产出
- 政策阶段划分（基于发文量/时间节点）
- 工具结构评估（各类别均衡性）
- 主体协同分析（单独发文vs联合发文）
- 目标覆盖评估（全面性）
- 时序演化规律
- 政策力度量化
- 政策协同性分析

## 第七步：撰写报告

**阅读 `references/报告撰写规范.md` 获取学术报告标准格式。**

### 报告结构
1. **摘要**：研究问题、样本、核心发现
2. **描述性统计**：文本分布、类型、发文机构
3. **框架分析**：单维度与交叉维度结果（附表/图）
4. **核心发现**：3-5项主要发现及证据支撑
5. **局限**：样本范围、方法约束
6. **建议**：基于分析的政策优化建议

### 质量检查清单
- [ ] 编码信度达到阈值
- [ ] 框架有理论根基
- [ ] 发现由数据引用支撑
- [ ] 交叉维度洞察揭示非显性规律
- [ ] 建议具体可操作
- [ ] 遵循学术引用规范
