## 一、语料处理原则

### 原文优先

- 以政策发布机构的原文为语义判断依据。
- 官方译本用于辅助理解和报告表达，但仍需核对关键条款原文。
- 非官方译文只能作为工作译文，不作为最终证据来源。

### 版本记录

为每份政策建立元数据：

| 字段 | 说明 |
|---|---|
| policy_id | 样本编号 |
| original_title | 原文标题 |
| translated_title | 译名 |
| language | zh / en / bilingual / other |
| jurisdiction | 国家、地区或国际组织 |
| issuing_body | 发文机构 |
| policy_type | 法律、法规、strategy、guidance、regulation 等 |
| publication_date | 发布日期 |
| version_status | 原文、官方译本、研究者译文、修订版 |
| source_url | 官方来源 |

### 不直接合并近义词

跨语种分析时，不要仅凭字面相似合并概念。例如：

| 英文术语 | 常见中文译法 | 编码注意 |
|---|---|---|
| regulation | 规制、监管、法规 | 需区分法律文本、监管行为和政策工具 |
| guidance | 指引、指导意见 | 在不同制度中约束力差异很大 |
| accountability | 问责、责任机制 | 需区分政治问责、行政责任和绩效责任 |
| compliance | 合规、遵从 | 需区分企业合规、机构执行和个人义务 |
| subsidy | 补贴、资助 | 需区分直接财政补贴、税收优惠、隐性扶持 |

## 二、双语术语表

在正式编码前建立术语表，至少包含：

| 字段 | 示例 |
|---|---|
| term_original | accountability |
| term_translation | 问责 |
| definition | 对责任主体履责情况进行监督、评价和追责的制度安排 |
| coding_category | 保障机制/监督问责 |
| include | audit requirement, reporting obligation |
| exclude | general responsibility statement without enforcement |
| example_original | The agency shall report annually to Parliament. |
| example_translation | 该机构应每年向议会报告。 |
| note | 英国语境中 Parliament reporting 不等同于中国行政问责 |

### 命名规则

- 代码名可使用双语并列：`regulatory control / 规制管制`。
- 一级类别保持稳定，二级类别允许按制度语境扩展。
- 术语表更新时记录版本，避免前后编码口径漂移。

## 三、跨语种编码

### 统一代码本

中文和英文语料使用同一套核心代码本，但允许设置语境特有类别。

示例：

| 核心类别 | 中文表达 | 英文表达 | 说明 |
|---|---|---|---|
| 供给型工具 | 财政投入、人才培养、平台建设 | funding, capacity building, infrastructure | 政府直接提供资源 |
| 环境型工具 | 标准制定、法规管制、规划引导 | standards, regulation, strategic guidance | 塑造制度环境 |
| 需求型工具 | 政府采购、消费补贴、示范应用 | public procurement, demand incentives, pilot adoption | 拉动需求或应用场景 |

### 分析单元

- 中文政策默认以句子、分句、条款为单位。
- 英文政策默认以 sentence、clause、section、article 为单位。
- 长句多重义务并列时，按独立政策举措拆分。
- 对跨语种比较，分析单元粒度必须一致：不能中文按条款、英文按段落直接比较频次。

### 信度检验

跨语种研究在常规 Kappa / Alpha 外增加：

1. **术语一致性检查**：抽查同一术语在不同政策中的编码是否一致。
2. **双语复核**：由双语编码员复核关键条款和边界案例。
3. **回译抽查**：对研究者译文抽取 10%-20% 进行 back-translation 或同伴复核。
4. **分语种信度报告**：分别报告中文子样本、英文子样本和合并样本的一致性。

## 四、跨国比较

### 比较顺序

1. 比较政策问题是否等值：是否处理同一公共问题。
2. 比较制度功能是否等值：相似术语是否承担相似治理功能。
3. 比较政策工具组合：供给、环境、需求或强制、混合、自愿。
4. 比较执行机制：责任主体、资金来源、监督机制、评估机制。
5. 比较话语框架：问题定义、价值取向、风险叙事、目标群体。

### 必须记录的语境变量

| 变量 | 说明 |
|---|---|
| jurisdiction | 国家/地区/国际组织 |
| legal_system | 大陆法系、普通法系、欧盟法、国际组织规则等 |
| governance_level | 中央、联邦、州/省、市、国际组织 |
| instrument_tradition | 规制导向、市场激励导向、规划导向等 |
| implementation_style | 强制执行、协商治理、地方分权、试点扩散等 |

### 不可比情形

以下情况应说明限制，而不是强行比较：

- 两国政策处理的问题边界不同。
- 同名工具法律约束力不同。
- 一方为正式法律，另一方只是战略白皮书。
- 一方有完整执行细则，另一方只有原则性目标。
- 译文无法确认关键义务、责任主体或适用范围。

## 五、报告与引用

### 中文报告引用英文政策

推荐格式：

> 该政策要求监管机构进行年度报告（"report annually"），体现出持续评估与责任追踪机制。

原则：
- 正文使用中文解释政策含义。
- 保留英文关键词或短语，便于读者追溯。
- 自译长句时标注“作者译”。

### 英文报告引用中文政策

推荐格式：

> The document emphasizes "whole-process supervision" (全过程监督), indicating a shift from ex ante approval to continuous regulatory monitoring.

Principles:
- Explain the policy meaning in English.
- Preserve key Chinese terms when they carry institution-specific meaning.
- Mark researcher translations as "author's translation" when no official translation exists.

### 参考文献

- 中文政策按 GB/T 7714 或目标期刊要求处理。
- 英文政策按 APA、Chicago 或目标期刊要求处理。
- 同一政策有原文和官方译本时，优先引用原文；译本可作为附加来源说明。
