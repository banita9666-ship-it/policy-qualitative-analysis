## 一、编码前准备

### 编码团队组建

| 角色 | 人数 | 要求 |
|---|---|---|
| 主编码员 | 1-2人 | 熟悉政策领域，理解分析框架 |
| 独立编码员 | 1-2人 | 具备社会科学研究方法训练 |
| 仲裁员 | 1人（备选） | 资深研究者，处理分歧 |

### 编码手册编制

编码手册必须包含：各维度定义、类别/子类别界定（附正反例）、边界案例规则、不编码标准、多维度编码规则、编码格式规范。

### 编码培训流程

1. **独立阅读**：所有编码员各自阅读5-10份样本政策
2. **集体研讨**：讨论编码决策，统一理解
3. **试编码**：独立对2-3份政策进行试编码
4. **差异讨论**：比较编码差异，分析原因
5. **手册修订**：根据试编码结果完善编码手册

### 预编码（Pilot Coding）

正式编码前必须进行预编码：
- 抽取总样本的10%-15%
- 独立编码后计算一致性
- 若未达标（Kappa < 0.8），需：
  - 分析分歧类别
  - 修订类别定义
  - 补充边界案例
  - 重新培训
  - 再次预编码

## 二、编码规则

### 分析单元选择

| 单元类型 | 说明 | 适用场景 |
|---|---|---|
| 句子 | 单个语法完整的句子 | 最常用；边界清晰 |
| 分句 | 句子内的独立分句 | 一个句子包含多个政策举措时 |
| 段落 | 完整段落 | 语境依赖性强时 |
| 条款 | 政策条/款/项 | 结构性分析 |

**默认规则**：以句子为基本单元。当一个句子包含多个独立政策举措时，拆分为分句分别编码。

### 编码格式

标准格式：`政策编号-章-节-条-款`

```
示例：
第3份政策，第2章，第5条，第1款 → 3-2-5-1
第10份政策，第1章，第3条，第2款 → 10-1-3-2
```

简化格式（当结构层次不多时）：`政策编号-条-款`

```
示例：第5份政策，第12条，第3款 → 5-12-3
```

### 多维度编码规则

当一个文本单元涉及多个维度时：
- 在不同维度上分别编码
- **同一维度内不重复计数**
- 记录所有编码及编码理由

**示例**：
```
文本："卫生健康委应当建立基层医生培训制度，所需经费纳入市级财政预算。"

工具维度：供给型 → 人力资源供给 → 人才培训
目标维度：服务体系 → 基层能力建设
主体维度：政府机构 → 卫生健康委
力度维度：强制要求（"应当"）
```

### 编码决策树

```
该文本单元是否涉及具体政策举措？
├── 否 → 不编码
└── 是 → 涉及哪些维度？
    ├── 工具维度？
    │   ├── 什么类型？供给型/环境型/需求型
    │   └── 什么子类型？（资金、法规、培训等）
    ├── 目标维度？
    │   └── 哪个目标领域？
    ├── 主体维度？
    │   └── 涉及哪些利益相关方？
    ├── 力度维度？
    │   └── 强制性程度如何？
    └── 过程维度？
        └── 属于哪个政策阶段？
```

### 不编码标准

以下内容不纳入编码：
- 纯行政程序性内容（发文日期、生效日期、废止条款）
- 无实质内容的序言/前言
- 纯粹修辞性表述
- 已废止或被替代的条款
- 仅引用其他政策而无新增内容的条款
- 术语定义（除非定义本身体现政策取向）

## 三、信效度检验

### 信度检验方法

#### 方法一：百分比一致率

```
一致率 = (编码一致数 / 编码总数) × 100%

阈值标准：
- ≥ 95%：优秀
- ≥ 90%：良好
- ≥ 85%：可接受
- < 85%：需重新培训或修订框架
```

#### 方法二：Cohen's Kappa系数

```
Kappa = (P_o - P_e) / (1 - P_e)

其中：
P_o = 观察到的一致比例
P_e = 期望的随机一致比例
```

| Kappa值 | 一致程度 |
|---|---|
| < 0.20 | 轻微 |
| 0.21-0.40 | 一般 |
| 0.41-0.60 | 中等 |
| 0.61-0.80 | 高度 |
| 0.81-1.00 | 几乎完美 |

**学术发表可接受阈值**：Kappa ≥ 0.80

#### 方法三：Krippendorff's Alpha

适用于多名编码员、多类别编码场景：

```
Alpha = 1 - (D_o / D_e)

其中：
D_o = 观察到的不一致
D_e = 期望的随机不一致
```

**可接受阈值**：Alpha ≥ 0.80

**推荐**：当编码员超过2人或类别数较多时，优先使用Krippendorff's Alpha。

### 信度检验操作流程

1. 两名编码员独立编码相同子样本（10%-20%）
2. 构建编码决策列联表
3. 计算百分比一致率
4. 计算Cohen's Kappa（或Krippendorff's Alpha）
5. 若未达阈值：
   a. 逐条分析分歧编码
   b. 识别问题类别（计算类别级Kappa）
   c. 修订类别定义和边界案例
   d. 针对性再培训
   e. 重新检验直至达标

### 效度保障策略

#### 内容效度
- 框架维度来源于文献综述和理论
- 专家评议（CVI ≥ 0.8）
- 预编码检验框架的覆盖性

#### 三角验证

三角验证是提高研究可信度的重要策略：

| 三角验证类型 | 操作方式 |
|---|---|
| 方法三角 | 混合使用质性和量化方法 |
| 数据源三角 | 比较不同来源的政策文本 |
| 研究者三角 | 多名研究者独立分析 |
| 理论三角 | 运用多种理论视角解读 |

### 信效度报告模板

```
信效度检验报告
─────────────────────────────────
样本信息
  总样本量：[N]份，检验子样本：[n]份（[X]%）
  分析单元：句子/分句/条款
  编码维度/类别数：[N]个/[N]个

编码员：[编码员1]、[编码员2]

检验结果
  一致率：[X]%  |  Cohen's Kappa：[X.XX]
  Krippendorff's Alpha：[X.XX]（编码员>2人时使用）

正式编码：两名编码员分别完成全部样本
─────────────────────────────────
```

## 四、常见问题处理

### 问题1：语义模糊文本

**场景**：文本含义不明确，可能归入多个类别

**处理**：
1. 优先归入最具体的适用类别
2. 若确实无法判断，记录为"未编码"并说明理由
3. 若此类情况频繁出现，需修订类别定义

### 问题2：多功能政策举措

**场景**：一个举措同时服务多个目标

**处理**：
- 应用"不可再分原则"——编码其最主要的目的
- 判断依据：上下文语境、措辞强调、资源分配
- 若确实多重目的同等重要，可在不同目标维度分别编码

### 问题3：政策语言的时代演变

**场景**：同一术语在不同时期含义不同

**处理**：
- 建立时代特定的编码注释
- 例如"信息化"在2000年代指基础设施，在2020年代指数字转型
- 在编码手册中注明关键术语的历时演变

### 问题4：引用条款

**场景**：大量引用其他政策文件

**处理**：
- 若引用条款本身无新增实质内容 → 不编码
- 若引用条款结合本政策新增具体要求 → 编码新增部分
- 单独记录"引用密度"作为分析指标

### 问题5：特定类别分歧集中

**处理流程**：
1. 计算各类别单独Kappa值
2. 识别Kappa < 0.70的"问题类别"
3. 重新审视该类别的定义
4. 增加边界案例示例
5. 针对问题类别进行额外培训
6. 重新检验

## 五、NVivo操作指南

### 编码比较工作流程

1. 创建项目，导入所有政策文本
2. 建立编码结构（Nodes→New Node，为每个维度创建节点层级）
3. 分配文档给不同编码员独立编码
4. 运行编码比较：Query→Coding Comparison
5. 导出一致/分歧报告，自动计算Kappa系数

### 核心功能速查

| 功能 | 操作路径 |
|---|---|
| 创建节点 | 右键Nodes→New Node |
| 编码 | 选中文本→右键→Code Selection |
| 编码比较 | Query→Coding Comparison |
| 撰写备忘录 | 右键文档→Memo |
| 矩阵查询 | Query→Coding Matrix |
