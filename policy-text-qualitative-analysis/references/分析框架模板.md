## 一、政策工具分类体系

### 分类A：Rothwell-Zegveld（产业/科技政策推荐）

| 类型 | 说明 | 子工具 |
|---|---|---|
| **供给型** | 政府直接提供资源 | 资金投入、基础设施建设、人才培养、技术支持、信息服务 |
| **环境型** | 营造有利环境 | 法规管制、标准制定、税收优惠、战略规划、法规保障 |
| **需求型** | 拉动市场需求 | 政府采购、示范项目、服务外包、贸易促进、消费补贴 |

### 分类B：Howlett-Ramesh（政府干预程度）

| 类型 | 说明 | 子工具 |
|---|---|---|
| **强制型** | 具有约束力的命令 | 法律、法规、指令、禁令、强制标准 |
| **混合型** | 强制与自愿结合 | 信息公开、自愿标准、公私合作、认证制度 |
| **自愿型** | 市场或自我规制 | 行业自律、社区项目、市场激励、志愿参与 |

### 分类C：McDonnell-Elmore（机制导向）

| 类型 | 说明 | 子工具 |
|---|---|---|
| **命令型** | 要求遵从的义务性规定 | 规则、标准、许可、认证 |
| **激励型** | 以绩效为条件的财务转移 | 拨款、补贴、合同、税收减免 |
| **能力建设型** | 人力与智力资本投资 | 培训、技术援助、信息服务 |
| **系统变革型** | 改变权力/资源分配格局 | 分权、私有化、重组 |

### 分类D：中国特色政策工具（党建/治理维度）

| 类型 | 说明 | 子工具 |
|---|---|---|
| **党建引领型** | 党组织领导与统筹 | 党委领导、党建考核、党员先锋、组织动员 |
| **规划部署型** | 中长期战略规划 | 五年规划、专项规划、行动计划、实施方案 |
| **试点推广型** | 渐进式政策扩散 | 改革试点、示范区、经验推广、典型引路 |
| **督查考核型** | 监督与绩效管理 | 督察巡查、目标考核、问责机制、绩效评估 |

## 二、多维分析框架

### 标准三维框架：工具 × 目标 × 主体

**X维——政策工具**：从上述分类A/B中选择

**Y维——政策目标**（按领域定制）：

示例——教育政策目标：
- 教育公平与均衡发展
- 教育质量提升
- 人才培养与队伍建设
- 教育信息化与技术创新
- 教育治理现代化

示例——环境政策目标：
- 污染防治与减排
- 生态保护与修复
- 资源节约与循环利用
- 气候变化应对
- 环境监管能力建设

示例——科技政策目标：
- 基础研究强化
- 关键核心技术攻关
- 科技成果转化
- 创新主体培育
- 创新生态优化

**Z维——政策主体**：

| 主体类别 | 典型机构 |
|---|---|
| 党的领导机关 | 中共中央、中央纪委、中央各领导小组 |
| 中央行政机关 | 国务院、各部委 |
| 地方行政机关 | 省级/市级政府及职能部门 |
| 事业单位 | 学校、科研院所、公立医院 |
| 市场主体 | 国有企业、民营企业、行业协会 |
| 社会组织 | 社会团体、基金会、民办非企业 |
| 公众 | 公民、家庭、社区 |

### 替代三维框架：主体 × 路径 × 工具

适用于跨部门/融合类政策（如文旅融合、医养结合、产学研协同）：

**X维——政策主体**：发文机构
**Y维——融合路径**：
- 制度融合（体制机制创新）
- 产业融合（产业链整合）
- 产品融合（跨界产品服务）
- 服务融合（一体化服务供给）
- 技术融合（技术交叉应用）
- 空间融合（物理空间整合）

**Z维——政策工具**：标准分类

### 四维度扩展：工具 × 目标 × 主体 × 力度

**W维——政策力度**（量化评估维度）：

| 力度层级 | 说明 | 评分 |
|---|---|---|
| 5-战略引领 | 国家战略/党中央国务院层面 | 5 |
| 4-强力推动 | 国务院专项/多部委会签 | 4 |
| 3-部署落实 | 部委层面/省级政府 | 3 |
| 2-指导协调 | 司局层面/工作指导 | 2 |
| 1-参考倡导 | 地方/行业自律/建议性 | 1 |

**政策力度加权计算**：
```
单份政策力度 = 发文机构级别得分 × 0.4 + 政策类型得分 × 0.3 + 措施强制性得分 × 0.3
```

### 政策过程分析框架

适用于分析政策生命周期：

| 阶段 | 核心问题 | 分析重点 |
|---|---|---|
| 议程设置 | 问题如何进入政策议程？ | 政策背景、触发事件、问题界定 |
| 政策制定 | 政策如何被制定？ | 制定主体、决策程序、利益博弈 |
| 政策执行 | 政策如何被执行？ | 执行机制、资源配置、协调安排 |
| 政策评估 | 政策效果如何评价？ | 评估指标、反馈机制、调整条款 |

### PMC得分计算

```
一级变量得分: X_i = (该变量下二级得分之和) / (二级变量个数)

PMC指数: PMC = (所有一级变量得分之和) / (一级变量个数)

示例：
X1有4个二级变量，全部得1分 → X1 = 4/4 = 1.0
9个一级变量总分7.2 → PMC = 7.2/9 = 0.80 → 换算为8.0/10
```

### PMC评级

| PMC得分 | 评级 | 说明 |
|---|---|---|
| 9.0-10.0 | 完美 | 全面均衡的高质量政策 |
| 7.0-9.0 | 优秀 | 质量良好，存在轻微不足 |
| 5.0-7.0 | 可接受 | 基本合格，存在明显缺陷 |
| 0-5.0 | 较差 | 存在重大缺口，需大幅修订 |

### PMC可视化
- **PMC曲面图**：3×3矩阵展示各变量强弱
- **雷达图**：多维度对比
- **条形图**：政策间横向对比

```
因果条件 ──→ 现象 ──→ 脉络
                │
                ↓
中介条件 ──→ 行动策略 ──→ 结果
```

           [范畴A: 因果条件]
                  ↓

| 策略类型 | 说明 | 示例 |

## 六、自定义框架构建指南

当现有框架不适用时：

### 构建流程
1. **文献综述**：梳理同类政策研究使用的维度，记录领域特定因素
2. **专家咨询**：采用2-3轮德尔菲法确定维度，内容效度指数CVI ≥ 0.8
3. **框架初稿**：撰写维度定义与子类别，编制编码手册并附示例
4. **预编码测试**：用10-15%样本试编码，基于涌现模式修正类别
5. **框架定稿**：正式编码前锁定框架，记录所有修改

### 框架质量标准
- [ ] **互斥性**：各类别之间无重叠
- [ ] **完备性**：覆盖所有可能情况
- [ ] **理论根基**：有文献或理论支撑
- [ ] **经验可证**：可通过经验数据验证
- [ ] **操作可行**：实际编码时可区分
- [ ] **粒度适中**：不过于笼统也不过于细碎
